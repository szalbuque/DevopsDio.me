Ferramentas de colaboração

- Fork: o repositório fica sempre ligado ao repositório de origem.
- Clone: baixar o repositório para a máquina local
- Branch: criar uma branch para trabalhar no código
- Fazer o commit das alterações.
- Push: fazer o push da nova branch.
- O próprio github já sugere a url para fazer o pull request
- Explicar os detalhes do pull request
- O dono do repositório original pode fazer um merge

======================
Permissões

- No repositório > settings > manage access
- Invite a collaborator
- Vai chegar um e-mail para o convidado
- Ele aceita
- O colaborador vai poder fazer push diretamente no repositório
